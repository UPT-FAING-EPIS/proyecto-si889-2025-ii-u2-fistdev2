**Resumen del Laboratorio PETI 01: Despliegue de un Entorno de Desarrollo**

**Puntos Clave**

El informe del "Laboratorio PETI 01", presentado por Akhtar Oviedo Ahmed Hasan para el curso "Planeamiento Estratégico de TI" de la Universidad Privada de Tacna, detalla el proceso práctico de configuración y despliegue de un entorno de desarrollo. Los objetivos principales se centraron en aplicar un procedimiento de despliegue, configurar la conexión entre un servidor web y una base de datos, seguir instrucciones técnicas de un archivo README y desarrollar competencias en depuración y resolución de problemas. La práctica, con una duración de dos horas académicas, utilizó herramientas como un IDE (Entorno de Desarrollo Integrado), el gestor de bases de datos HeidiSQL y Python como lenguaje de ejecución.

**Hallazgos**

El procedimiento, guiado por el Dr. Oscar J. Jimenez-Flores, consistió en descomprimir los archivos del proyecto, abrirlos en un IDE y seguir los pasos indicados en la documentación. Un paso fundamental fue la creación de una base de datos en HeidiSQL y la configuración de las credenciales de conexión. Durante el proceso, se encontraron dos errores críticos que impidieron el funcionamiento inicial. El primer error se produjo porque no se habían importado las tablas ni los datos a la base de datos, lo que impedía que el servidor se levantara correctamente. Este fue resuelto cargando los scripts necesarios en la base de datos. Un segundo error requirió la modificación del código de hasheo de la contraseña para evitar fallos de autenticación. Tras solucionar ambos problemas, se logró ejecutar el servidor web con éxito, demostrando una correcta comunicación entre los componentes del sistema.

**Recomendaciones**

La práctica subraya varias lecciones clave. Primero, destaca la importancia de seguir rigurosamente la documentación técnica (README) y prestar atención a los detalles, como la correcta configuración de credenciales, para evitar errores comunes. Segundo, evidencia la necesidad crítica de asegurar que la base de datos esté correctamente inicializada y poblada con tablas y datos antes de ejecutar la aplicación, confirmando la fuerte dependencia entre el backend y el almacenamiento de información. Finalmente, la experiencia refuerza que la habilidad de depuración y *troubleshooting* es esencial para diagnosticar y resolver eficazmente los problemas que surgen durante la implementación de soluciones TI en entornos prácticos.
